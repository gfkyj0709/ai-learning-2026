FDS 머신러닝 모델 설계

1. 사용 모델 종류

1-1. 이상치 탐지 모델 (Anomaly Detection)
- Isolation Forest: 비지도 학습 기반, 정상 데이터로만 학습하여 이상 거래 탐지
- Autoencoder: 딥러닝 기반 이상치 탐지, 재구성 오류가 큰 거래를 이상으로 분류
- One-Class SVM: 정상 거래의 경계를 학습하여 경계 밖 거래 탐지

1-2. 분류 모델 (Classification)
- XGBoost / LightGBM: 레이블된 사기 거래 데이터로 학습, 높은 정확도
- 랜덤 포레스트: 다양한 특성 조합으로 앙상블 예측, 해석 용이
- 딥러닝 (LSTM): 시계열 거래 패턴 학습으로 순차적 이상 탐지

2. 주요 입력 피처
- 거래 금액, 거래 시각(시간대, 요일)
- 최근 N일 평균 거래 금액 대비 비율
- 접속 IP 국가 코드, 기기 지문(Device Fingerprint)
- 수취인 계좌 신뢰도 점수
- 최근 로그인 실패 횟수
- 계좌 개설 후 경과일수
- 동일 수취인 거래 빈도

3. 모델 성능 지표
- Precision (정밀도): 이상 탐지 예측 중 실제 이상 비율
- Recall (재현율): 실제 이상 거래 중 탐지 비율 (높을수록 중요)
- F1 Score: Precision과 Recall의 조화 평균
- AUC-ROC: 전반적인 모델 판별 능력 평가
- FPR (False Positive Rate): 정상 거래 오탐 비율 (낮을수록 좋음)

4. 모델 운영 전략
- 실시간 스코어링: 거래 발생 시 200ms 이내 점수 산출
- 모델 재학습: 월 1회 이상 신규 사기 패턴 반영
- A/B 테스트: 신규 모델과 기존 모델 병렬 운영 후 성능 비교
- 설명 가능성(XAI): SHAP 값으로 탐지 근거 제공 (규제 대응)

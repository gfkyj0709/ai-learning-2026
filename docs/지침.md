# 설계 패턴 / DB 전략 학습 범위

## Phase 2에서 학습
- **CQRS** (Command Query Responsibility Segregation)
  - 읽기/쓰기 모델 완전 분리
  - @Transactional(readOnly=true) → 서브DB, @Transactional → 마스터DB
  - 실무 연결: FDS Engine(쓰기) / AI Assistant 조회(읽기) 분리 구조
- **Outbox Pattern**
  - DB 저장 + Kafka 발행을 원자적으로 처리
  - 실무 연결: FDS Engine → Kafka → Forwarder 구조의 안전한 Spring 구현
- **AbstractRoutingDataSource** (Read/Write Splitting)
  - 설정파일 기반 마스터/서브 DB 자동 라우팅
  - 실무 연결: 직접 구현했던 패턴을 Spring 방식으로 재학습

## Phase 3에서 학습
- **Saga Pattern**: MSA 환경에서 분산 트랜잭션 처리
- **Redis Cache**: @Cacheable + FDS 룰 캐싱

# Capacity Planning (용량 산정)

## 기본 흐름
- 예상 동시 사용자 수 (DAU/MAU) 파악

## FDS 관점 예시
- 하루 거래 100만건 기준
- 피크타임 집중 (낮 12~1시, 저녁 7~9시)
- 피크 TPS = 100만 / (2시간 * 3600초) ≈ 140 TPS
- 여유분 포함 200 TPS 대응 가능하게 설계

## 서비스별 자원 특성
- 메모리 집중: Redis (데이터 전부 메모리), Kafka (버퍼링), JVM Spring Boot (힙)
- 디스크 집중: PostgreSQL (데이터 + WAL 로그), Qdrant (벡터), Kafka (메시지 보존)
- CPU 집중: LLM 추론 (Ollama), 임베딩 생성, 암호화 처리

## 메모리 사전 할당 설정

### JVM (Spring Boot .jar)
- -Xms: 시작 시 최소 힙 메모리 (미리 할당)
- -Xmx: 최대 힙 메모리 (초과 불가)
- Dockerfile 예시: ENTRYPOINT ["java", "-Xms512m", "-Xmx1g", "-jar", "app.jar"]

### PostgreSQL (postgresql.conf)
- shared_buffers: DB 캐시용 메모리 (전체 RAM의 25% 권장)
- work_mem: 쿼리당 정렬/해시 메모리 (기본 4MB)
- max_connections: 최대 커넥션 수
- effective_cache_size: OS 캐시 포함 가용 메모리 예상값

### Redis (redis.conf)
- maxmemory: 최대 메모리 한도 (예: 512mb)
- maxmemory-policy: 한도 초과 시 제거 정책 (allkeys-lru 권장)

### Docker Compose 레벨 설정
- deploy.resources.limits.memory: 컨테이너 최대 메모리 제한
- deploy.resources.reservations.memory: 최소 보장 메모리

# 최종 프로젝트 — FDS AI 어시스턴트
- 프로젝트명: FDS AI Assistant
- 핵심 아이디어: FDS 점검 데이터/룰 문서를 AI가 분석하여 자연어 및 음성으로 질의응답
- 활용 목적: 실무 도구 + 포트폴리오 + 유튜브 콘텐츠 시리즈 ("FDS 솔루션에 AI를 붙여봤다")
- 목표 완성 시점: Phase 3 (2026년 9~11월)

기술 스택 매핑:
- Spring Cloud MSA: Gateway + 각 기능을 마이크로서비스로 분리
- RAG + Qdrant: FDS 점검 보고서/룰 문서 벡터 저장 및 검색
- AI Agent (LangGraph): 이상탐지 패턴 자동 분석 및 리포트 생성
- Whisper STT: 음성 질문 입력 처리 ("이번 달 이상탐지 몇 건?")
- AWS or GCP + Docker: 전체 서비스 배포 인프라
- PostgreSQL + Qdrant: 정형 데이터 + 벡터 데이터 이원화 저장

# 코드 작성 규칙
- 코드 작성 시 반드시 주석을 함께 작성해줘
- 클래스/메서드 상단에는 JavaDoc 주석 작성 (@param, @return 포함)
- 모든 라인에 주석 작성 (새로운 유지보수 담당자가 바로 이해하고 수정할 수 있는 수준)
- 핵심 로직은 간결하게, 나머지는 꼼꼼하게 설명해줘
- 주석은 영어 먼저, 한국어 병기 (영어로 먼저 읽고 한국어로 확인)

주석 예시:
/**
 * Validates the user authentication token.
 * 사용자 인증 토큰을 검증한다.
 *
 * @param token JWT token to validate / 검증할 JWT 토큰
 * @return Whether the token is valid / 유효 여부
 */
public boolean validateToken(String token) {
    // Return false immediately if token is null or empty
    // 토큰이 null이거나 비어있으면 즉시 false 반환
    if (token == null || token.isEmpty()) {
        return false;
    }
}
